# Tasks

Registro de tarefas executadas no projeto, organizadas por data.

Cada subpasta representa um dia de trabalho e contém checklists, decisões e validações realizadas durante a execução.

---

## Formato

```
tasks/
└── YYYY-MM-DD/
    └── CHECKLIST.md    ← tarefas do dia, verificações e resultados
```

## Convenções

- Cada CHECKLIST.md deve referenciar os documentos relevantes (DESIGN-SYSTEM.md, UI-PATTERNS.md, MAPEAMENTO-TELAS.md, etc.)
- Ao concluir itens, marcar com `[x]`
- Resultados de verificação (`tsc --noEmit`, build, etc.) devem ser registrados
